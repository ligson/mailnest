# 邮件界面、搜索过滤与规则归档设计

## 1. 背景与目标

当前邮件页采用表格 + 右侧抽屉的方式查看邮件，适合早期功能验证，但不像真实邮箱客户端，长时间使用时信息密度、浏览效率和分类入口都不理想。

本设计目标是把 Mail Nest 的邮件页升级为三栏邮箱客户端：

- 左侧：邮箱导航、系统文件夹、自定义文件夹和规则入口。
- 中间：邮件列表、搜索框和过滤条件。
- 右侧：邮件详情阅读区、附件区和基础操作。

同时设计本地规则归档能力：用户可以配置匹配规则，把符合条件的邮件放入 Mail Nest 本地文件夹。规则只影响本地展示与分类，不修改远端 IMAP 邮箱中的文件夹。

## 2. 范围

### 2.1 本阶段实现范围

- 三栏式邮件主界面。
- 三栏宽度支持桌面端拖拽调整，便于在不同屏幕宽度下平衡文件夹、列表和阅读区空间。
- 邮件列表搜索与过滤：
  - 关键词搜索：主题、发件人、收件人、抄送、正文。
  - 发件人邮箱过滤。
  - 主题过滤。
  - 日期范围过滤。
  - 是否有附件过滤。
  - 邮箱账号过滤。
  - 本地文件夹过滤。
- 邮件详情固定在右侧阅读区，不再依赖抽屉作为主要阅读方式。
- 本地文件夹数据模型与基础接口。
- 规则数据模型与基础接口。
- 邮箱账号和规则支持在 Web 界面中编辑。
- 邮件列表和详情接入联系人通讯录，优先显示联系人昵称或姓名，并可点击查看真实邮箱地址和联系信息。
- 新邮件收取完成后自动执行规则。
- 支持手动对历史邮件重新应用规则。

### 2.2 暂不实现范围

- 不移动远端 IMAP 服务器上的邮件。
- 不同步远端 IMAP 文件夹。
- 不实现全文搜索引擎；MVP 使用 SQLite `LIKE` 和已保存正文文件的轻量索引字段。
- 不实现复杂布尔表达式，例如任意嵌套的 AND/OR。
- 不实现规则动作链，例如同时标记已读、转发、删除。

## 3. 用户体验设计

### 3.1 三栏布局

```text
┌────────────────┬────────────────────────────┬──────────────────────────────┐
│ 左侧导航        │ 中间邮件列表                │ 右侧邮件详情                  │
├────────────────┼────────────────────────────┼──────────────────────────────┤
│ 收件箱          │ 搜索框                      │ 主题                          │
│ 全部邮件        │ 过滤条：账号/日期/附件       │ 发件人、收件人、时间            │
│ 有附件          │ 邮件列表                    │ HTML/纯文本正文                │
│ 自定义文件夹     │ 分页或加载更多              │ 附件列表与下载                  │
│ 规则管理        │                            │                              │
└────────────────┴────────────────────────────┴──────────────────────────────┘
```

### 左侧导航

- 系统文件夹：
  - 收件箱：默认展示 `INBOX` 下邮件。
  - 全部邮件：展示当前用户全部邮件。
  - 有附件：展示带附件邮件。
- 自定义文件夹：
  - 用户手动创建。
  - 规则动作可选择这些文件夹。
- 规则管理入口：
  - 进入规则列表。
- 新增、启用、禁用、删除规则。
- 新增、编辑、启用、禁用、删除规则。
  - 手动应用规则。

### 中间列表

邮件列表不再使用传统表格，而是使用邮箱风格列表项：

- 第一行：发件人、时间、附件图标。
- 第二行：主题。
- 第三行：正文摘要。
- 选中邮件使用左侧高亮线和浅色背景。
- 发件人和收件人显示优先级为通讯录昵称、通讯录姓名、邮件头显示名、邮箱用户名部分，避免列表直接堆满邮箱地址。

列表顶部提供：

- 搜索框。
- 日期范围选择。
- 账号选择。
- 附件筛选。
- 更多筛选按钮。

### 右侧详情

右侧详情常驻：

- 未选择邮件时显示空状态。
- 选择邮件后展示主题、发件人、收件人、时间、所属文件夹、正文和附件。
- 发件人、收件人和抄送人使用联系人标签展示，点击标签弹出真实邮箱地址、电话、公司和备注。
- 附件以紧凑列表展示，包含文件名、类型、大小、下载按钮。
- HTML 正文继续使用后端处理过的内嵌图片。

## 4. 搜索与过滤设计

### 4.1 查询参数

邮件列表接口扩展为：

```text
GET /api/v1/messages
```

支持查询参数：

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `page` | number | 页码 |
| `pageSize` | number | 每页数量 |
| `accountId` | string | 邮箱账号 ID |
| `folderId` | string | 本地文件夹 ID |
| `systemFolder` | string | 系统文件夹：`inbox`、`sent`、`all`、`attachments` |
| `keyword` | string | 关键词，匹配主题、发件人、收件人、抄送和正文索引 |
| `from` | string | 发件人过滤 |
| `subject` | string | 主题过滤 |
| `dateFrom` | string | 起始日期，格式 `YYYY-MM-DD` |
| `dateTo` | string | 结束日期，格式 `YYYY-MM-DD` |
| `hasAttachments` | boolean | 是否有附件 |

### 4.2 正文搜索策略

MVP 不引入全文搜索引擎，采用轻量索引字段：

- 在 `mail_messages` 中增加 `body_preview` 或 `search_text` 字段。
- 收取邮件时从纯文本正文或 HTML 去标签文本中提取前若干字符，写入该字段。
- `keyword` 使用 SQLite `LIKE` 查询 `subject`、`from_addr`、`to_addrs`、`cc_addrs`、`search_text`。

后续如果邮件量变大，再考虑 SQLite FTS5。

## 5. 文件夹设计

### 5.1 系统文件夹

系统文件夹不需要单独入库：

- `inbox`：`mail_messages.folder = 'INBOX'`。
- `sent`：`mail_messages.folder` 等于邮箱账号配置的发件箱文件夹名。
- `all`：当前用户全部邮件。
- `attachments`：`has_attachments = 1`。

### 5.2 自定义文件夹

新增表 `mail_folders`：

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `user_id` | 用户 ID |
| `name` | 文件夹名称 |
| `color` | 可选颜色 |
| `sort_order` | 排序 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

在 `mail_messages` 中增加：

| 字段 | 说明 |
| --- | --- |
| `local_folder_id` | 本地文件夹 ID，可为空 |

约束：

- 文件夹必须按 `user_id` 隔离。
- 同一用户下文件夹名称不允许重复。
- 删除文件夹时，将相关邮件的 `local_folder_id` 置空，不删除邮件。

## 6. 规则设计

### 6.1 数据模型

新增表 `mail_rules`：

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `user_id` | 用户 ID |
| `name` | 规则名称 |
| `enabled` | 是否启用 |
| `match_mode` | `all` 或 `any`，表示条件全部满足或任一满足 |
| `target_folder_id` | 命中的邮件放入哪个本地文件夹 |
| `sort_order` | 执行顺序 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |

新增表 `mail_rule_conditions`：

| 字段 | 说明 |
| --- | --- |
| `id` | 主键 |
| `rule_id` | 规则 ID |
| `field` | 匹配字段 |
| `operator` | 匹配操作符 |
| `value` | 匹配值 |

### 6.2 匹配字段

MVP 支持：

- `from`：发件人。
- `from_domain`：发件人邮箱域名。
- `to`：收件人。
- `cc`：抄送。
- `subject`：主题。
- `body`：正文索引。
- `has_attachments`：是否有附件。

### 6.3 操作符

MVP 支持：

- `contains`：包含。
- `equals`：等于。
- `starts_with`：开头匹配。
- `ends_with`：结尾匹配。
- `domain_equals`：域名等于，仅用于 `from_domain`。
- `is_true`：为真，仅用于 `has_attachments`。
- `is_false`：为假，仅用于 `has_attachments`。

### 6.4 执行策略

- 新邮件插入成功后执行当前用户启用的规则。
- 规则按 `sort_order` 升序执行。
- MVP 中一封邮件只归属一个本地文件夹。
- 如果多个规则命中，采用第一条命中的规则。
- 手动重新应用规则时，可以选择：
  - 只处理未归档邮件。
  - 重新处理全部邮件并覆盖现有本地文件夹。

## 7. API 设计

所有接口仍使用统一 envelope。

### 7.1 文件夹接口

```text
GET /api/v1/mail-folders
POST /api/v1/mail-folders
PUT /api/v1/mail-folders/{id}
DELETE /api/v1/mail-folders/{id}
```

创建请求：

```json
{
  "name": "安全通知",
  "color": "#1f66d1",
  "sortOrder": 10
}
```

### 7.2 规则接口

```text
GET /api/v1/mail-rules
POST /api/v1/mail-rules
PUT /api/v1/mail-rules/{id}
DELETE /api/v1/mail-rules/{id}
POST /api/v1/mail-rules/apply
```

创建请求：

```json
{
  "name": "安全通知归档",
  "enabled": true,
  "matchMode": "all",
  "targetFolderId": "1",
  "sortOrder": 10,
  "conditions": [
    {
      "field": "subject",
      "operator": "contains",
      "value": "网络安全"
    },
    {
      "field": "has_attachments",
      "operator": "is_true",
      "value": ""
    }
  ]
}
```

手动应用请求：

```json
{
  "scope": "unfiled"
}
```

`scope` 可选值：

- `unfiled`：只处理未归档邮件。
- `all`：重新处理全部邮件并覆盖现有文件夹。

## 8. 前端页面设计

### 8.1 邮件页

组件建议：

- `MailShellView.vue`：三栏主页面。
- `MailFolderNav.vue`：左侧系统文件夹和自定义文件夹。
- `MailSearchBar.vue`：搜索框和过滤条件。
- `MailList.vue`：邮箱风格列表。
- `MailReader.vue`：邮件详情阅读区。
- `AttachmentList.vue`：附件列表。

### 8.2 规则管理页

可以先作为邮件页中的侧边面板或独立路由 `/rules`。

MVP 推荐独立路由：

- 规则列表。
- 新建/编辑规则抽屉。
- 条件编辑器。
- 目标文件夹选择。
- 启用/禁用开关。
- 手动应用规则按钮。

## 9. 测试重点

后端：

- 搜索过滤必须按 `user_id` 隔离。
- 关键词搜索覆盖主题、发件人、收件人、抄送和正文索引。
- 日期范围过滤边界正确。
- 文件夹查询不能越权。
- 规则执行只影响当前用户邮件。
- 多条规则命中时只使用第一条。
- 删除文件夹不删除邮件。

前端：

- 无邮件时显示空状态。
- 选择邮件后右侧详情正确刷新。
- 过滤条件变更后列表刷新。
- 附件邮件可以正常下载附件。
- 小屏幕下三栏可以降级为列表 + 详情抽屉或上下布局。

## 10. 分阶段实施建议

### 阶段 A：邮件页视觉和基础搜索

- 后端扩展 `GET /api/v1/messages` 查询参数。
- 增加 `search_text` 字段并在收取时写入。
- 前端改造为三栏布局。
- 支持关键词、账号、日期、附件过滤。

### 阶段 B：本地文件夹

- 新增 `mail_folders` 表和接口。
- `mail_messages` 增加 `local_folder_id`。
- 左侧展示自定义文件夹。
- 支持按文件夹过滤。

### 阶段 C：规则归档

- 新增规则表和规则条件表。
- 实现规则匹配服务。
- 收取新邮件后自动执行规则。
- 支持手动重新应用规则。
- 前端增加规则管理页。

## 11. 设计约束

- 所有新增接口必须使用统一 envelope。
- 所有查询必须带当前登录用户上下文。
- 规则匹配不能读取其他用户邮件。
- 文件夹和规则数据必须包含 `user_id` 或可追溯到用户。
- 前端不直接读取本地文件，附件仍通过后端鉴权接口下载。
