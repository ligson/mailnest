# 垃圾邮件系统文件夹与规则标记设计

## 1. 目标

Mail Nest 增加本地“垃圾邮件”系统文件夹，用于把广告、钓鱼、低价值通知等邮件从收件箱和默认列表中隔离出来。垃圾邮件能力优先服务本地 Web 查看和整理，不直接修改远端 IMAP 服务器上的邮件状态或目录。

## 2. 数据模型

在 `mail_message_states` 中增加：

| 字段 | 说明 |
| --- | --- |
| `is_spam` | 是否被标记为垃圾邮件 |
| `spam_at` | 最近一次标记为垃圾邮件的时间 |

垃圾邮件状态仍按 `user_id + message_id` 隔离和唯一。标记或取消垃圾邮件只更新状态表，不修改 `mail_messages` 原始元数据、正文文件、附件文件和远端邮箱。

## 3. 查询与展示

- `GET /api/v1/messages?systemFolder=spam` 返回当前用户垃圾邮件。
- 默认邮件列表、收件箱、本地文件夹和附件视图排除 `is_spam = 1` 的邮件。
- 垃圾邮件邮件仍可打开详情、下载附件和继续执行删除、已读、星标等本地操作。

## 4. 批量操作

批量接口新增动作：

- `mark_spam`：将选中邮件标记为垃圾邮件。
- `unmark_spam`：从垃圾邮件移出，恢复到原始 IMAP 文件夹或本地文件夹视图。

批量预览返回 `spamCount`，用于展示选中邮件中已有垃圾邮件数量。

## 5. 规则引擎

规则动作新增 `mark_spam`。当新邮件入库或用户手动“应用到历史邮件”时，规则命中即可把邮件标记为垃圾邮件。

`mark_spam` 不需要目标文件夹；只有 `move_folder` 动作需要 `targetFolderId`。规则仍按优先级和 `stopOnMatch` 执行。

## 6. 安全与部署

垃圾邮件迁移只新增状态字段，不删除用户数据。上线前需要备份远端 `docker-compose.yml`、`config.yaml` 和持久化 `data/` 目录；上线后验证健康检查、登录、邮件列表、垃圾邮件视图和规则保存可用。
