# Mail Nest 实施计划

## 1. 总体顺序

建议分阶段实现，先完成可运行闭环，再扩展高级能力。

一期已确认决策：

- 登录态使用 JWT。
- 只支持 IMAP/IMAPS。
- 收取 `INBOX` 和可配置的发件箱文件夹。
- 邮箱密码或授权码加密后存入 SQLite。
- 是否开放注册由后端 `config.yaml` 控制。

## 2. 阶段一：项目骨架与基础约定

目标：建立后端、前端基础工程，并落实统一响应格式。

任务：

1. 创建 `mailnest-be/` Go 项目。
2. 创建 `mailnest-fe/` Vue 3 + Vite + TypeScript 项目。
3. 后端实现 `config.yaml` 配置加载、本地数据目录初始化、SQLite 连接。
4. 后端实现统一 envelope 响应封装。
5. 前端实现 API 客户端和 envelope 解包逻辑。
6. 补充基础 README 和启动说明。

验收：

- 后端健康检查接口可访问。
- 健康检查接口返回统一 envelope。
- 前端可以调用健康检查接口并展示基础页面。

## 3. 阶段二：用户注册登录

目标：实现用户系统和登录态。

任务：

1. 设计并迁移 `users` 表。
2. 实现密码哈希。
3. 实现由 `config.yaml` 控制的注册接口。
4. 实现登录接口。
5. 实现当前用户接口。
6. 实现 JWT 生成和鉴权中间件。
7. 实现前端登录、注册页面。
8. 实现前端路由鉴权。

验收：

- 用户可以注册并登录。
- 未登录用户访问业务页面会被跳转到登录页。
- 登录后可以获取当前用户信息。

## 4. 阶段三：邮箱账号配置

目标：用户可以配置多个邮箱账号。

任务：

1. 设计并迁移 `mail_accounts` 表。
2. 实现邮箱凭据加密和解密。
3. 实现邮箱账号增删改查接口。
4. 实现 IMAP 连接测试接口。
5. 实现前端邮箱账号列表和编辑表单。
6. 实现连接测试和启停状态展示。
7. 增加 SMTP 发信配置字段，密码或授权码按邮箱凭据规则加密保存。

验收：

- 一个用户可以创建多个邮箱账号。
- 邮箱密码或授权码不会明文落库。
- 用户无法访问其他用户的邮箱账号。

## 5. 阶段四：邮件收取

目标：实现 IMAP 邮件拉取和本地存储。

任务：

1. 设计并迁移 `mail_messages`、`mail_attachments`、`mail_sync_jobs` 表。
2. 实现 IMAP 收件箱拉取。
3. 实现邮件 MIME 解析。
4. 实现原始邮件、正文和附件本地落盘。
5. 实现邮件去重。
6. 实现手动触发收取接口。
7. 实现后台周期性收取 worker：每分钟轻量扫描到期启用账号，按账号收取间隔触发日常增量收取，并限制并发避免拖慢服务。
8. 实现收取任务状态记录。

验收：

- 用户可以手动触发某个邮箱收取邮件。
- 启用账号到达收取间隔后可以由后端自动触发日常增量收取。
- 收取完成后数据库中出现邮件元数据。
- 本地目录中保存邮件原文、正文和附件。
- 重复收取不会重复插入同一封邮件。

## 6. 阶段五：邮件 Web 查看

目标：用户可以通过 Web 查看邮件。

任务：

1. 实现邮件列表接口。
2. 实现邮件详情接口。
3. 实现附件下载接口。
4. 实现前端邮件列表页面。
5. 实现前端邮件详情页面。
6. 实现按邮箱账号过滤和基础分页。

验收：

- 用户可以查看自己的邮件列表。
- 用户可以打开邮件详情查看正文。
- 用户可以下载附件。
- 用户无法访问其他用户的邮件详情或附件。

## 7. 阶段六：联系人通讯录

目标：用户可以维护联系人，并在邮件列表和详情中优先显示联系人昵称或姓名。

任务：

1. 设计并迁移 `contacts` 表，按 `user_id + email_key` 去重。
2. 实现联系人列表、新增、编辑、删除接口。
3. 邮件收取时从发件人、收件人和抄送人自动沉淀联系人。
4. 自动沉淀只更新最近出现时间和空缺显示名，不覆盖用户手工维护资料。
5. 前端新增联系人页面，支持搜索、分页和维护资料。
6. 邮件列表和详情按邮箱地址匹配通讯录，优先显示昵称或姓名，点击联系人可查看真实邮箱地址和联系信息。

验收：

- 用户可以维护自己的联系人。
- 用户无法访问其他用户联系人。
- 自动发现联系人不会覆盖手工昵称、电话、公司或备注。
- 邮件列表和详情能优先显示联系人名称，详情中可查看完整邮箱地址。

## 8. 阶段七：邮件发送

目标：用户可以通过 Web 界面发送邮件，并在本地已发送目录中立即查看。

任务：

1. 设计并迁移邮箱账号 SMTP 配置字段。
2. 实现 SMTP/SMTPS/STARTTLS 发信服务，支持纯文本和 HTML 正文。
3. 实现 `POST /api/v1/messages/send` 接口，按当前用户校验发件账号归属。
4. 发送成功后保存 RFC822 原文、正文和邮件元数据到本地已发送目录。
5. 发信后自动沉淀收件人、抄送人和密送人为联系人。
6. 前端邮件页新增写邮件抽屉，支持选择发件账号和填写收件人、抄送、密送、主题、正文。

验收：

- 未登录用户不能发信。
- 用户不能使用其他用户的邮箱账号发信。
- SMTP 密码或授权码不会明文落库或回显。
- 发送成功后可以在“发件箱”看到本地保存的邮件。
- 发信相关联系人自动进入通讯录。

## 9. 阶段八：完善与加固

目标：提升稳定性、安全性和可维护性。

任务：

1. 增加关键单元测试和集成测试。
2. 完善错误提示和日志。
3. 增加收取任务并发控制。
4. 增加配置示例。
5. 补充部署说明。
6. 检查并更新全部文档。

验收：

- 核心测试通过。
- 常见错误能给出明确提示。
- 文档能指导新开发者启动项目。

## 10. 优先测试点

- 统一 JSON envelope。
- 用户注册、登录、鉴权。
- 用户数据隔离。
- 邮箱凭据加密。
- IMAP 连接测试。
- 邮件去重。
- 附件路径安全。
- 联系人 CRUD 和用户隔离。
- 自动沉淀联系人不覆盖手工维护资料。
- SMTP 发信配置加密、发信后本地保存和联系人沉淀。

## 11. 后续可扩展方向

- 常见邮箱配置向导和错误提示。
- POP3 支持。
- OAuth 邮箱授权。
- 发信附件、回复、转发和草稿。
- 邮件搜索。
- 标签和规则。
- 邮件导出。
- 管理员页面。
- Docker 部署。
