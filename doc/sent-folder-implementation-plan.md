# 发件箱查看实施计划

## 目标

让 Mail Nest 支持同步和查看已发送邮件。首期采用可配置发件箱 IMAP 文件夹名的方式，默认使用 `Sent`，并在邮件页左侧显示“发件箱”。

## 改动范围

- 后端数据：`mail_accounts` 增加 `sent_folder` 字段，默认 `Sent`。
- 后端同步：手动收取和全量同步同时同步 `INBOX` 与 `sent_folder`，保存邮件时使用真实系统文件夹名。
- 后端查询：`systemFolder=sent` 映射为账号配置的发件箱文件夹；`systemFolder=inbox` 保持 `INBOX`。
- 后端目录：提供账号级 IMAP 文件夹列表接口，识别 `\Sent` 属性和常见“已发送”目录名，供前端选择。
- 前端账号：新增/编辑邮箱账号时可配置“发件箱文件夹名”；编辑已有账号时可读取服务器目录并选择真实发件箱目录。
- 前端邮件页：系统文件夹增加“发件箱”。
- 容错策略：账号配置的发件箱目录不存在时跳过该目录并提示，不让 `INBOX` 同步失败。
- 文档和变更记录：更新 `CHANGELOG.md`，必要时同步 API/架构文档。

## 验证

- 后端测试覆盖同步发件箱和系统文件夹过滤。
- 前端构建通过。
- 线上部署前必须备份真实配置、数据库和数据目录。
