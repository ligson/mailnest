# Mail Nest 产品需求说明

## 1. 背景

Mail Nest（信匣）希望提供一个本地可部署的邮件收取与查看工具。用户通过 Web 界面管理多个邮箱账号，系统在后端收取邮件并保存到本地，用户可以在浏览器中查看邮件内容。

## 2. 一期目标

一期优先完成一个可用的最小闭环：

1. 用户可以注册、登录和退出登录。
2. 登录用户可以新增、编辑、删除自己的邮箱账号配置。
3. 一个用户可以配置多个邮箱。
4. 系统可以通过 IMAP/IMAPS 收取邮箱邮件。
5. 用户可以在 Web 界面查看自己的邮件列表和邮件详情。
6. 用户数据必须隔离，不同用户之间不能互相看到邮箱配置或邮件。
7. 所有接口使用统一 JSON envelope。
8. 用户可以维护联系人通讯录，邮件列表和详情优先显示联系人昵称或姓名，点击联系人后可查看具体邮箱地址和联系信息。
9. 用户可以通过已配置 SMTP 的邮箱账号发送邮件，发送成功后本地保存到已发送目录。

已确认的一期默认决策：

- 登录态使用 JWT。
- 邮件收取一期只支持 IMAP/IMAPS。
- 邮件发送一期支持 SMTP/SMTPS/STARTTLS、HTML 富文本正文、发信附件和账号签名模板，暂不支持草稿。
- 邮件文件夹一期必须收取 `INBOX` 和可配置的发件箱文件夹；发件箱目录应支持从 IMAP 服务器目录列表中选择，避免不同服务商目录名不一致导致无法同步已发送邮件。
- 只要邮件已经保存在邮箱服务器的收件箱或已发送目录中，Mail Nest 必须能通过手动收取、定时收取或全量同步保存到本地。
- 邮箱密码或授权码需要加密后存入数据库。
- 是否允许注册由后端 `config.yaml` 配置控制。

## 3. 一期范围

### 3.1 用户系统

- 注册：使用用户名、邮箱、密码注册。
- 注册开关：是否允许开放注册由后端 `config.yaml` 控制。
- 登录：使用用户名或邮箱加密码登录。
- 退出登录：清理登录态。
- 当前用户：前端可获取当前登录用户基础信息。
- 密码保存：必须使用安全哈希，禁止明文保存。

### 3.2 邮箱账号管理

- 新增邮箱账号配置。
- 修改邮箱账号配置。
- 删除邮箱账号配置。
- 启用或停用邮箱账号。
- 测试邮箱连接。
- 查看邮箱账号最近一次收取状态。

邮箱账号配置建议包含：

- 显示名称。
- 邮箱地址。
- IMAP 主机。
- IMAP 端口。
- 是否使用 TLS。
- 登录用户名。
- 邮箱密码或授权码。
- SMTP 主机、端口、SSL/TLS 或 STARTTLS、登录用户名、密码或授权码。
- 收取间隔。
- 是否启用。

### 3.3 邮件收取

- 支持手动触发某个邮箱账号收取。
- 支持后台周期性收取。
- 支持收取并保存账号的收件邮件和已发送邮件；不能只保存 `INBOX`。
- 收取过程需要记录状态、开始时间、结束时间、错误信息和新增邮件数。
- 同一邮箱同一文件夹内邮件需要去重。
- 如果本地客户端发送邮件但未同步到服务器发件箱目录，后端无法通过 IMAP 获取，需要用户在客户端开启保存已发送邮件到服务器。

### 3.4 邮件查看

- 邮件列表：展示主题、发件人、收件人、日期、所属邮箱、是否有附件等。
- 邮件详情：展示主题、发件人、收件人、抄送、日期、正文、附件列表。
- 发件人、收件人和抄送人展示应优先使用通讯录昵称，其次使用联系人姓名或邮件头中的显示名，避免列表中大量暴露邮箱地址。
- 邮件详情中的联系人标签可点击查看完整邮箱地址、电话、公司和备注等通讯录信息。
- 正文：需要支持纯文本和 HTML。
- 附件：一期可先支持查看附件元数据和下载，预览能力后续扩展。

### 3.5 联系人通讯录

- 用户可以新增、编辑、删除自己的联系人。
- 联系人至少包含邮箱地址、昵称、姓名、电话、公司、备注、来源和最近出现时间。
- 联系人按 `user_id + email` 隔离和去重，不同用户的通讯录互不可见。
- 邮件收取时可自动从发件人、收件人和抄送人中沉淀联系人；自动沉淀只能更新最近出现时间和空缺显示名，不能覆盖用户手工维护的昵称、姓名、电话、公司和备注。
- 邮件列表和详情根据联系人邮箱地址做本地匹配，优先显示用户维护的昵称或姓名；仍必须能点击查看真实邮箱地址。

### 3.6 邮件发送

- 用户可以在邮件页打开写邮件抽屉，选择自己的邮箱账号作为发件账号。
- 支持填写收件人、抄送、密送、主题和纯文本正文。
- 发送前必须校验邮箱地址格式，且至少存在一个收件人、抄送人或密送人。
- 后端使用账号保存的 SMTP 配置发信，SMTP 密码或授权码必须加密落库，不得在响应或日志中输出明文。
- SMTP 发送成功后，后端应把该邮件保存为本地已发送邮件，使用账号配置的发件箱目录名，方便在“发件箱”中立即查看。
- 发信过程中出现的收件人、抄送人和密送人应自动沉淀为联系人；自动沉淀仍不能覆盖用户手工维护资料。

## 4. 非一期范围

以下能力暂不作为一期必需交付：

- 邮件回复、转发、草稿。
- 发信附件。
- OAuth 邮箱授权。
- 全文搜索引擎。
- 复杂邮件规则、自动标签和过滤器。
- 多端实时同步。
- 企业级权限、组织和角色管理。
- 云端对象存储。

## 5. 角色与权限

一期只有普通用户角色：

- 用户只能管理自己的邮箱账号。
- 用户只能查看自己的邮件。
- 用户只能触发自己的邮箱收取任务。

后续可以扩展管理员角色，用于查看系统状态和管理用户。

## 6. 成功标准

- 新用户可以完成注册、登录。
- 用户可以配置两个以上邮箱账号。
- 用户可以手动触发收取，并在收取完成后看到邮件列表。
- 用户 A 无法通过接口或前端访问用户 B 的邮件和邮箱配置。
- 前端所有接口调用都能按统一 envelope 处理成功和失败。
- 重要变更被记录到 `CHANGELOG.md`。
